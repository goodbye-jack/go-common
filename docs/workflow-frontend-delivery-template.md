# 工作流前端接入交付模板

## 1. 文档用途

这份文档是给业务系统前端开发组直接执行的交付模板。

目标不是讲通用理念，而是明确前端应该：

- 建哪些页面
- 调哪些接口
- 展示哪些字段
- 如何展示流程图和时间线

## 2. 前端接入目标

前端接入完成后，至少需要提供下面这些页面能力：

1. 业务创建页
2. 我的申请页
3. 我的待办页
4. 我的已办页
5. 任务处理页
6. 流程图查看能力
7. 时间线查看能力

## 3. 前端职责边界

### 3.1 前端必须负责

- 业务页面渲染
- 表单录入
- 流程图内嵌展示
- 时间线展示
- 当前办理人或候选范围展示
- 任务处理提交

### 3.2 前端不要负责

- 自己拼接工作流时间线逻辑
- 自己解析 BPMN XML
- 自己猜当前办理人
- 依赖 Flowable 官方表单页面做业务处理界面

## 4. 最小页面清单

### 4.1 创建页

职责：

- 录入业务字段
- 提交业务创建请求
- 发起流程

### 4.2 我的申请页

职责：

- 展示当前人发起的业务单
- 展示业务状态和流程状态
- 支持查看流程图
- 支持查看时间线

### 4.3 我的待办页

职责：

- 展示当前登录人的待办
- 显示当前节点
- 显示当前办理人或候选范围
- 支持进入任务处理页
- 列表中只展示轻量流程摘要，不在列表里加载完整流程图和完整时间线

### 4.4 我的已办页

职责：

- 展示当前登录人的已办
- 展示完成时间
- 支持查看流程图和时间线
- 列表中只展示已办节点和轻量流程摘要

### 4.5 任务处理页

职责：

- 展示任务上下文
- 展示业务数据
- 展示节点表单参考
- 提交处理结果

## 5. 前端最小接口清单

前端最少需要接下面这些接口：

- `POST /api/applications`
- `GET /api/applications`
- `GET /api/todo`
- `GET /api/done`
- `GET /api/tasks/{taskId}/context`
- `POST /api/tasks/{taskId}/complete`
- `GET /api/process-instances/{id}/progress-view`
- `GET /api/process-instances/{id}/progress-timeline`
- `GET /api/biz/{bizId}/progress-view`
- `GET /api/biz/{bizId}/progress-timeline`
- `GET /api/process/instance/{id}/definition-xml`

## 6. 推荐页面结构模板

### 6.1 我的申请页结构

建议模块：

1. 业务基础信息
2. 流程状态摘要
3. 当前节点和当前人员
4. 内嵌流程图
5. 流程时间线

### 6.2 待办页结构

建议模块：

1. 待办列表
2. 已办列表
3. 每条记录的轻量流程摘要
4. 查看流程进度的展开面板

### 6.3 任务处理页结构

建议模块：

1. 任务基础信息
2. 业务数据区
3. 流程状态区
4. 流程图区
5. 时间线区
6. 处理表单区

## 7. BPMN 流程图展示要求

流程图展示必须遵守以下规则：

- 流程图必须内嵌在页面中
- 不允许图内滚动
- 整张图必须在容器内完整显示
- 已完成节点、当前节点、未来节点需要有清晰区分

推荐实现：

- 使用 BPMN XML 渲染组件
- 页面滚动，而不是图内滚动
- 通过 `fit-viewport` 思路做自适应

## 8. 时间线展示要求

时间线至少要展示：

- 节点名称
- 节点状态
- 办理人
- 候选人
- 候选组
- 开始时间
- 结束时间

推荐状态文案：

- `STARTED` -> 已发起
- `ACTIVE` -> 处理中
- `PENDING` -> 未开始
- `COMPLETED` -> 已完成
- `TERMINATED` -> 已终止

需要特别说明：

- 列表页只展示轻量流程摘要
- 完整时间线只能在详情区、展开区或处理页按需加载
- 不要在列表首屏批量请求所有流程时间线

## 9. 人员展示规则

前端必须统一采用下面这套规则：

1. 如果有 `assignee`，显示真实办理人
2. 如果没有 `assignee`，但有 `candidateUsers` 或 `candidateGroups`，显示“待认领”
3. 如果是非人工节点，才显示“系统”

推荐展示文案：

- `办理人：张三`
- `待认领 / 候选人：张三 / 候选组：部门审批组`
- `系统`

## 10. 业务标题展示要求

标题必须在下面这些位置都能看到：

- 我的申请
- 我的待办
- 我的已办
- 任务详情

如果某个地方标题没显示出来，前端应先确认：

1. 接口响应里是否有 `title`
2. 页面是否正确渲染了 `title`

不要用任务节点名去替代业务标题。

## 11. 表单引用展示要求

Flowable 表单模型只作为参考，不作为业务页面生成依据。

前端建议展示：

- 节点是否配置了表单
- 表单字段列表
- 表单 outcome 列表

但真正的业务录入控件仍然由前端自己实现。

## 12. 任务处理页字段设计建议

建议前端任务处理页统一保留以下字段区：

- 处理结果
- 处理意见
- 业务扩展字段
- 特殊节点字段，例如是否需要专家复核

要求：

- 页面字段命名清晰
- 提交前统一映射到后端需要的格式

## 13. 联调顺序

前端联调建议按下面顺序推进：

1. 先打通创建页
2. 再打通我的申请
3. 再打通我的待办
4. 再打通任务详情
5. 再打通任务处理提交
6. 再打通我的已办
7. 最后处理流程图和时间线细节

## 14. 前端组交付物

前端组提测前，必须提供：

- 页面清单
- 路由清单
- 接口调用清单
- 页面截图
- 一组完整流程的录屏或截图

## 15. 验收标准

前端接入完成必须满足：

- 标题显示完整
- 当前节点显示正确
- 当前办理人或候选范围显示正确
- 流程图可正常内嵌展示
- 时间线顺序正确
- 处理页可正常提交

## 16. 前端开发任务拆解模板

前端组可以直接按下面的粒度拆任务。

### 16.1 任务一：创建页

需要完成：

- 表单录入
- 提交创建接口
- 成功后跳转我的申请或详情页

完成标准：

- 创建成功
- 标题和主字段显示正确

### 16.2 任务二：我的申请页

需要完成：

- 列表查询
- 展示标题、状态、当前节点、当前人员
- 支持查看流程图和时间线

完成标准：

- 当前人能看到自己发起的业务
- 进度展示入口可用

### 16.3 任务三：待办和已办页

需要完成：

- 待办列表
- 已办列表
- 进入任务处理页
- 查看进度

完成标准：

- 标题显示不丢失
- 完成时间显示正确

### 16.4 任务四：任务处理页

需要完成：

- 任务上下文加载
- 业务数据展示
- 节点表单参考展示
- 提交处理结果

完成标准：

- 一次处理能够提交成功
- 下一环节能看到新待办

### 16.5 任务五：流程图和时间线体验优化

需要完成：

- 流程图内嵌
- 自适应展示
- 时间线状态文案和人员文案统一

完成标准：

- 页面滚动正常
- 图内无滚动
- 人员显示不误导

## 17. 页面字段清单模板

前端组建议为每个页面整理一份字段清单，避免联调时字段遗漏。

### 17.1 我的申请页字段

- `bizId`
- `title`
- `status`
- `createdAt`
- `currentNodeName`
- `currentAssignees`
- `currentCandidateUsers`
- `currentCandidateGroups`

### 17.2 待办页字段

- `taskId`
- `taskName`
- `title`
- `bizId`
- `createdAt`
- `processInstanceId`
- `currentNodeName`

### 17.3 已办页字段

- `taskId`
- `taskName`
- `title`
- `bizId`
- `completedAt`
- `processInstanceId`

### 17.4 任务处理页字段

- `taskId`
- `activityId`
- `processInstanceId`
- `bizId`
- `title`
- `bizData`
- `variables`
- `formRef`

## 18. 前端联调截图清单

前端组提测前，建议至少准备下面这些截图：

1. 创建成功截图
2. 我的申请列表截图
3. 待办列表截图
4. 已办列表截图
5. 任务处理页截图
6. 流程图截图
7. 时间线截图

截图要求：

- 标题清晰可见
- 当前节点清晰可见
- 人员信息清晰可见

## 19. 常见前端误区

下面这些做法需要避免：

- 用节点名称代替业务标题
- 只展示“系统”，不展示真实办理人或候选范围
- 在流程图容器里加内部滚动条
- 自己从 BPMN XML 推断流程时间线
- 看到表单引用就想自动生成业务表单

## 20. 前端组提测前最终自查

前端负责人提测前建议逐项核对：

1. 创建页字段都能正常提交
2. 我的申请里标题和状态显示正确
3. 待办里标题、节点、人员显示正确
4. 已办里标题、完成时间显示正确
5. 任务详情能看见业务数据和表单参考
6. 流程图在页面里完整展示
7. 时间线顺序符合真实流转过程
8. 有候选人或候选组时不再误显示“系统”

满足以上条件，前端接入才算完成。
