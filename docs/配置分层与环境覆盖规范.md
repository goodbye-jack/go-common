# 配置分层与环境覆盖规范

## 1. 目标

统一约束 `config.yaml` 与 `config.<env>.yaml` 的职责边界，避免环境选择职责和配置值本身混在一起。

## 2. 基本原则

- `config.yaml` 是公共基线
- `config.dev.yaml / config.test.yaml / config.prod.yaml` 是差异覆盖
- `CONFIG_ENV` 是唯一环境选择器
- `CONFIG_FILE` 是显式文件选择器，优先级高于 `CONFIG_ENV`
- `app.env` 是运行时展示值，不承担环境选择职责

## 3. 加载顺序

1. 先读取 `config.yaml`
2. 如果存在 `CONFIG_FILE`，读取 `config.yaml + CONFIG_FILE`
3. 否则如果存在 `CONFIG_ENV`，读取 `config.yaml + config.<env>.yaml`
4. 两者都不存在时，仅读取 `config.yaml`

## 4. 覆盖文件禁止项

以下 key 不允许出现在环境覆盖文件中：

- `app.name`
- `app.env`

## 5. 同步到业务项目的规则产物

配置同步器会额外生成：

- `config.layering.yaml`
- `config.rules.md`

用途分别是：

- `config.layering.yaml`：机器可读的校验规则
- `config.rules.md`：给业务开发直接阅读的规范说明
