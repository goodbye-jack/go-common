# 配置项登记规范

## 1. 目标

为了避免 `go-common` 新增配置项后只有代码知道、模板和文档不知道，所有新增配置项都必须同步登记到 `configspec/`。

## 2. 什么时候必须登记

以下情况必须更新 `configspec`：

- 新增 `config.GetConfigString("...")`
- 新增 `config.GetConfigInt("...")`
- 新增 `config.GetConfigBool("...")`
- 新增 `viper.GetString("...")`
- 新增 `viper.GetInt("...")`
- 新增 `viper.GetBool("...")`
- 新增 `viper.GetStringMap(...)`
- 新增 `viper.GetStringMapString(...)`

## 3. 登记到哪里

按模块登记到：

- `configspec/modules/service.yaml`
- `configspec/modules/workflow.yaml`
- `configspec/modules/workflow_directory.yaml`
- `configspec/modules/workflow_assignment.yaml`
- `configspec/modules/databases.yaml`
- `configspec/modules/rbac_compatibility.yaml`

## 4. 登记最少要填哪些字段

每个配置项最少要有：

- `key`
- `kind`
- `since`
- `comment`

推荐同时补齐：

- `type`
- `default`
- `example`
- `group`
- `order`
- `merge_policy`

## 5. 密码类字段要求

如果配置项属于密码、token、secret：

- 必须标记 `sensitive: true`
- `example` 只能使用占位符
- 禁止写入真实环境值

示例：

```yaml
example: flowable-rest-password
```

## 6. 兼容字段要求

历史兼容配置项必须：

- 标记 `compatibility_only: true`
- 放入 `rbac_compatibility.yaml` 或对应兼容模块
- 不进入新项目主模板

## 7. 当前版本口径

当前模板体系以 `v1.3.5` 为准：

- 服务基础配置统一使用 `app.*`、`server.*`、`security.cookie.*`、`storage.local.*`
- 数据库不引入 `enabled`
- 新项目初始化模板默认使用 `MySQL + Redis`
