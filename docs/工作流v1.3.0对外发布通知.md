# 工作流 v1.3.0 对外发布通知

各业务开发组、测试组：

`go-common/workflow` 正式版 `v1.3.0` 已完成发布准备，当前版本可作为业务系统接入工作流平台的正式稳定版本使用。

本次正式版重点能力如下：

- 固定了工作流正式标准接口清单
- 统一了单流程与父子流程的流程图入口 `diagram-view`
- 稳定了待办、已办、任务上下文、任务处理能力
- 稳定了按业务单号查询流程进度和时间线能力
- 支持基于 LDAP 的当前用户、上级、部门查询
- 明确了 Flowable 表单模型只做节点字段参考，不直接承担业务前端渲染

本次版本对业务系统的正式接入要求如下：

1. 业务系统保留自己的业务表、业务详情页、业务处理页
2. 发起流程时必须稳定传入 `bizId`、`bizType`、`title`
3. 业务系统应封装自己的业务发起接口，内部调用 `POST /api/process/start`
4. 新系统统一使用 `GET /api/process-instances/{id}/diagram-view`
5. 接入前必须先阅读《工作流配置说明》

正式推荐文档如下：

- 《工作流配置说明》
- 《工作流平台标准接口说明》
- 《工作流业务接入指南》
- 《工作流后端接入交付模板》
- 《工作流前端接入交付模板》
- 《工作流测试验收模板》

正式推荐版本：

- `v1.3.0`

兼容说明：

- 老系统如果仍使用 `definition-xml` 或 `composite-diagram`，当前版本仍保留兼容
- 新接入系统不再建议继续依赖兼容接口

接入过程中如涉及复杂 `callActivity`、会签、多实例或特殊租户策略，请在接入前先单独评估，不建议直接按标准单流程口径硬套。
