# 工作流正式版发布说明模板

## 1. 使用说明

本文档用于在 `go-common/workflow` 正式发布时，直接作为版本发布说明正文模板使用。

建议发布时只替换以下内容：

- 版本号
- 发布日期
- 本次重点变更
- 升级影响
- 已知边界

## 2. 标题模板

建议标题：

`go-common 工作流平台 v1.3.0 正式发布说明`

## 3. 正文模板

### 3.1 版本信息

- 版本号：`v1.3.0`
- 发布日期：`2026-03-13`
- 版本定位：首个正式稳定版

### 3.2 本次发布摘要

本次版本完成了工作流平台标准接口正式收口，统一了单流程与父子流程图入口，稳定了待办已办隔离、任务上下文、表单引用解析与业务接入路径，可作为业务系统正式接入的稳定版本使用。

### 3.3 本次新增与收口能力

本次正式发布版本重点提供以下能力：

- 标准流程启动接口
- 我的待办与我的已办接口
- 任务上下文接口
- 表单引用解析接口
- 任务完成接口
- 进度视图接口
- 时间线接口
- 统一流程图接口 `diagram-view`
- LDAP 用户、上级、部门查询接口

### 3.4 正式推荐接口清单

- `POST /api/process/start`
- `GET /api/me/tasks/todo`
- `GET /api/me/tasks/done`
- `GET /api/tasks/{id}/context`
- `GET /api/tasks/{id}/form-ref`
- `POST /api/tasks/{id}/complete`
- `GET /api/process-instances/{id}/progress-view`
- `GET /api/process-instances/{id}/progress-timeline`
- `GET /api/biz/{bizId}/progress-view`
- `GET /api/biz/{bizId}/progress-timeline`
- `GET /api/process-instances/{id}/diagram-view`

### 3.5 接入建议

推荐业务系统采用“两层发起接口”模式：

- 业务系统自己的发起业务单接口负责写业务表和绑定表
- 工作流标准启动接口负责启动流程并返回 `processInstanceId`

推荐页面结构：

- 发起页
- 我的申请
- 我的待办
- 我的已办
- 业务详情页
- 业务处理页

推荐业务组发布前至少同步阅读以下文档：

- 《工作流平台标准接口说明》
- 《工作流配置说明》
- 《工作流业务接入指南》
- 《工作流后端接入交付模板》

### 3.6 与旧版本的兼容关系

当前保留以下兼容接口：

- `GET /api/process/instance/{id}/definition-xml`
- `GET /api/process-instances/{id}/composite-diagram`

说明：

- 新接入系统请统一使用 `diagram-view`
- 老系统可以在兼容期内继续保留旧调用方式

### 3.7 升级影响

本次版本对新系统是正式推荐版本，对旧系统的影响主要如下：

- 如果旧前端直接依赖 `definition-xml`，当前仍可继续使用
- 如果旧前端已经接入复合流程图，可逐步迁移到 `diagram-view`
- 如果业务系统之前没有区分“业务发起接口”和“工作流启动接口”，建议本次一并整理

### 3.8 已完成验证场景

当前已经完成联调验证的业务场景包括：

- 文物鉴定流程
- 文物告警流程
- 包含 `callActivity` 的父子流程图统一展示场景

### 3.9 已知边界

当前版本已知边界如下：

- 会签、多实例、深层嵌套 `callActivity` 仍属于后续增强范围
- 更复杂 EL 函数表达式仍属于后续增强范围
- 大规模场景下待办已办仍建议继续推进投影表和索引化优化

### 3.10 发布结论

当前版本已经达到正式版发布条件，建议作为 `go-common/workflow` 首个正式稳定版发布，并作为后续业务系统标准接入基线使用。
