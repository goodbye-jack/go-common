# 工作流正式版发布检查结果

## 1. 文档目的

本文用于记录 `go-common/workflow` 在正式发布前最后一轮检查清单的执行结果，并明确当前版本是否已经达到正式版发布条件。

本文不是“计划”，而是“执行结果”。

检查时间：

- 2026-03-13

建议发布版本：

- `v1.3.0`

## 2. 检查范围

本次检查覆盖以下四类内容：

- 标准接口面是否已经收口稳定
- 两类真实业务流程与一类父子流程是否已完成联调验证
- 配置、接入、发布文档是否已补齐到可交付状态
- 兼容接口、租户、表单引用、流程图统一入口是否已达到正式版可用标准

## 3. 核心链路检查结果

### 3.1 流程发起

检查结论：

- 已通过

确认结果：

- `POST /api/process/start` 已作为正式标准启动接口收口
- 业务系统封装发起接口后，能够先写业务表，再发起流程，再回写 `processInstanceId`
- `bizId`、`bizType`、`title` 已形成稳定传递链路

### 3.2 我的待办

检查结论：

- 已通过

确认结果：

- 当前登录用户能够正确查询自己的待办
- 候选用户、候选组场景已按当前用户过滤
- 待办标题、业务单号、流程摘要补齐逻辑已经稳定
- 默认分页和轻量列表语义已生效

### 3.3 我的已办

检查结论：

- 已通过

确认结果：

- 当前登录用户能够正确查询自己的已办
- 已办隔离不再串别人的历史处理记录
- 已办列表已具备标题、业务单号、完成时间、流程摘要补充能力

### 3.4 任务上下文与处理

检查结论：

- 已通过

确认结果：

- `GET /api/tasks/{id}/context` 的 `task`、`business`、`variables`、`formRef` 四段结构已稳定
- `POST /api/tasks/{id}/complete` 能正常驱动流程继续流转
- 业务页面可以基于任务上下文回填标题、业务单号、变量和表单参考

## 4. 目录能力与配置检查结果

### 4.1 LDAP 目录能力

检查结论：

- 已通过

确认结果：

- 当前用户信息、上级、部门查询能力已具备
- LDAP 能力已与工作流用户上下文集成
- 目录查询能力可作为其他业务系统通用复用能力提供

### 4.2 配置说明完备性

检查结论：

- 已通过

本轮已补齐的配置文档包括：

- 《工作流配置说明》
- 《工作流业务接入指南》
- 《工作流后端接入交付模板》

当前已经明确写清：

- `config.yaml` 的最小模板
- `config.yaml` 的增强模板
- Flowable REST、LDAP、表单元数据库、回调、上下文头的配置方式
- 除了 `config.yaml` 之外，业务系统还必须完成的注册动作和上下文注入动作
- Flowable 平台侧必须对齐的用户、组、表单 key、租户、子流程规则

## 5. 表单引用解析检查结果

检查结论：

- 已通过

确认结果：

- 节点未配置表单时，不影响流程处理
- 节点配置表单但 key 不一致时，会表现为 `configured=true`、`resolved=false`
- 节点配置表单且 key、租户、元数据库一致时，可稳定解析为 `resolved=true`
- 表单引用解析已明确使用独立 Flowable 元数据库配置，不再依赖业务库

## 6. 进度与流程图检查结果

### 6.1 进度接口

检查结论：

- 已通过

确认结果：

- `progress-view` 能返回步骤摘要、当前节点和当前处理人信息
- `progress-timeline` 能返回完整正向时间线，并保留已走分支步骤表能力
- 当前处理人、候选人、候选组展示语义已统一

### 6.2 流程图统一入口

检查结论：

- 已通过

确认结果：

- `GET /api/process-instances/{id}/diagram-view` 已成为正式统一入口
- 单流程图不受复杂流程能力影响
- 父子流程图场景已由统一接口承载
- 兼容接口 `definition-xml`、`composite-diagram` 仍保留，未误伤旧能力

## 8. 兼容性与稳定性检查结果

### 8.1 兼容接口

检查结论：

- 已通过

确认结果：

- `GET /api/process/instance/{id}/definition-xml` 仍保留
- `GET /api/process-instances/{id}/composite-diagram` 仍保留
- 新系统已明确统一推荐使用 `diagram-view`

### 8.2 性能基线

检查结论：

- 当前版本达到正式版基线要求

确认结果：

- 列表接口默认轻量化已生效
- 默认分页已生效
- 列表不再默认逐条做重型进度 enrich
- 详情页再查询 `progress-view`、`progress-timeline`、`diagram-view` 的分层方式已成立
- 待办投影、回调驱动读模型增强第一版已进入可用状态

说明：

- 大规模数据场景仍建议继续推进投影表和更强索引化方案
- 这属于后续性能增强，不影响当前版本进入正式发布
