# 目录同步模块说明

## 目标

`directorysync` 是 go-common 中新增的通用目录同步模块，用来解耦：

- 业务系统如何组织用户/部门/岗位数据
- 目录系统（当前是 LDAP）如何落地这些数据

这样各业务项目只需要实现 `Source`，即可复用统一的同步校验、执行顺序和报告结构。

## 组成

- `types.go`：标准记录与报告模型
- `source.go`：业务系统数据源接口
- `sink.go`：目录落地接口
- `validate.go`：部门/岗位/用户同步前校验
- `engine.go`：通用执行引擎
- `ldap_sink.go`：LDAP 落地实现

## 约束

1. `go-common` 不放业务字段，不依赖具体业务模型。
2. 业务侧如需扩展目录特性，可通过 `Attributes` 透传扩展属性。
3. 当前 LDAP Sink 约定支持的部门扩展属性：
   - `manager_user_id`
   - `manager_uid`

当业务侧传入上述属性时：
- 首轮部门创建会先忽略领导 DN，避免“部门先于用户创建”导致 LDAP 校验失败。
- 用户同步完成后，`engine` 会自动做一次部门关系刷新，把 `managerDN` 写回 LDAP。

## 适用方式

- `go-common`：保持通用
- `业务项目`：负责映射规则、权限策略、同步入口、同步历史
