# go-common 工作流平台 v1.3.0 正式发布说明

> 说明：本文件保留为 `v1.3.0` 首个正式稳定版发布说明归档。当前推荐业务项目使用 `v1.3.1`，请优先参考《工作流v1.3.1正式发布说明》和《工作流v1.3.1业务项目最小接入手册》。

## 1. 版本信息

- 版本号：`v1.3.0`
- 发布日期：`2026-03-13`
- 版本定位：首个正式稳定版

## 2. 发布摘要

本次版本完成了 `go-common/workflow` 的正式接口收口和接入体系定稿，统一了单流程与父子流程的流程图入口，稳定了待办、已办、任务上下文、表单引用解析、按业务单号查进度等核心能力，可以作为业务系统正式接入工作流平台的稳定版本使用。

## 3. 本次正式发布能力

本次版本正式提供以下标准能力：

- 标准流程启动接口 `POST /api/process/start`
- 我的待办接口 `GET /api/me/tasks/todo`
- 我的已办接口 `GET /api/me/tasks/done`
- 任务上下文接口 `GET /api/tasks/{id}/context`
- 任务表单参考接口 `GET /api/tasks/{id}/form-ref`
- 任务处理接口 `POST /api/tasks/{id}/complete`
- 流程进度视图接口
- 流程时间线接口
- 统一流程图接口 `GET /api/process-instances/{id}/diagram-view`
- LDAP 用户、上级、部门查询接口

## 4. 本次版本重点收口内容

### 4.1 标准接口面已经固定

当前工作流正式标准接口、兼容接口、稳定字段、增强字段已经在《工作流平台标准接口说明》和《工作流接口收口说明》中固定，不再建议业务系统绕过这套标准自行拼装工作流能力。

### 4.2 流程图入口已经统一

从这一版开始：

- 单流程图
- 父子复合流程图

统一走：

- `GET /api/process-instances/{id}/diagram-view`

调用方无需再自行判断应该取原始 XML 还是复合视图。

### 4.3 待办与已办隔离已经稳定

本次版本重点修复了以下问题：

- 待办误显示给非当前用户
- 已办误显示他人历史记录
- 候选用户、候选组场景过滤不稳定

当前列表能力已经达到正式版稳定要求。

### 4.4 表单引用解析已经解耦

当前表单引用解析已经明确：

- 使用独立 Flowable 元数据库配置
- Flowable 表单模型只作为节点字段参考
- 表单解析失败不会阻断流程流转

### 4.5 业务接入方式已经标准化

当前正式推荐业务系统采用“两层发起接口”方式：

- 业务系统自己的发起接口负责业务表、业务 JSON、附件、绑定关系
- 工作流标准接口负责启动流程并返回 `processInstanceId`

这条边界已经明确，不再建议业务前端直接裸调工作流启动接口。

## 5. 已完成验证场景

当前版本已经完成以下真实场景联调：

- 文物鉴定流程
- 文物告警流程
- 含 `callActivity` 的父子流程复杂场景

验证范围已覆盖：

- 发起
- 我的申请
- 待办
- 已办
- 任务上下文
- 表单引用解析
- 进度视图
- 时间线
- 流程图

## 6. 接入要求

业务系统正式接入前，至少应完成以下事项：

1. 在 `config.yaml` 中配置 `workflow.flowable.*`
2. 配置 `workflow.api.*`、`workflow.context.*`
3. 如启用表单引用解析，配置 `workflow.formref.db_instance`
4. 配置 LDAP 目录参数
5. 启动阶段注册 `workflowapi.NewDefaultModuleFromConfig()`
6. 保证登录态可解析 `userId`，租户中间件可解析 `tenantId`
7. 业务系统自己的发起接口内部调用 `POST /api/process/start`
8. 如果启用回调，确保 `POST /flowable/callback` 可被 Flowable 平台访问

详细配置方式请直接参考：

- 《工作流配置说明》
- 《工作流业务接入指南》
- 《工作流后端接入交付模板》

## 7. 兼容说明

当前仍保留以下兼容接口：

- `GET /api/process/instance/{id}/definition-xml`
- `GET /api/process-instances/{id}/composite-diagram`

兼容策略如下：

- 新系统统一使用 `diagram-view`
- 老系统允许在兼容期内继续使用旧接口
- 后续如需下线兼容接口，应另行发布迁移通知

## 8. 已知边界

当前版本已知边界如下：

- 会签、多实例、深层嵌套 `callActivity` 仍属于后续增强范围
- 更复杂 EL 函数表达式仍属于后续增强范围
- 超大规模待办、已办场景仍建议继续推进投影表与索引化优化

这些边界不影响当前版本作为正式稳定版发布。

## 9. 发布结论

综合接口收口、真实业务联调、配置说明、兼容策略和发布检查结果，当前版本已经达到正式发布条件。

建议：

- 正式发布 `v1.3.0`
- 作为 `go-common/workflow` 首个正式稳定版
- 作为后续业务系统统一接入的正式基线版本
