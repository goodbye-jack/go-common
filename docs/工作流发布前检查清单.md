# 工作流发布前检查清单

## 1. 文档目的

本文用于在发布 `go-common/workflow` 正式版本前，确认当前版本是否已经达到可交付、可接入、可回归、可运维的标准。

判断原则：

- 满足核心链路、隔离、文档、回归四类条件后，才能发布正式版本
- 如果仍需依赖手工补数据、临时 SQL、局部绕过逻辑，则不应发布正式版

## 2. 正式版本的达标项

发布正式版前，至少需要同时满足以下条件。

### 2.1 核心链路完整

- 登录态上下文能够稳定注入
- 业务系统能够正常发起流程
- `bizId` 与 `processInstanceId` 能稳定绑定
- 当前用户能够正确查到自己的待办
- 当前用户完成任务后能够正确查到自己的已办
- 业务详情页能够按 `bizId` 查看进度和流程图
- 任务处理页能够获取任务上下文、变量、表单参考
- 流程结束后业务状态和流程状态同步正确

### 2.2 权限与隔离正确

- 当前用户不会看到别人的待办
- 候选用户模式下，只有具备候选资格的人能看到待办
- 当前用户不会看到别人的已办
- 待办与已办过滤统一收敛在工作流平台内部
- `tenantId` 与 `systemCode` 范围校验不会误放行

### 2.3 进度与流程图正确

- `progress-view` 正确
- `progress-timeline` 正确
- `diagram-view` 对单流程正确
- `diagram-view` 对父子流程正确
- 单流程图不受复杂流程能力影响
- 当前处理人、候选人、候选组展示正确

### 2.4 文档与发布材料齐备

- 标准接口说明齐备
- 接口收口说明齐备
- 业务接入指南齐备
- 前后端接入模板齐备
- 测试验收模板齐备
- 版本变更说明齐备
- 发布说明模板齐备

## 3. 当前版本重点检查项

围绕这次正式版准备，必须重点确认以下内容。

### 3.1 流程发起

- `POST /api/process/start` 可用
- 业务系统自己的发起封装接口可用
- 启动后能返回有效 `processInstanceId`
- 绑定表回写正确

### 3.2 待办与已办

- 默认分页正常
- 默认轻量列表语义正确
- `includeProgress` 行为正确
- 待办和已办人员隔离正确

### 3.3 任务上下文

- `task`
- `business`
- `variables`
- `formRef`

以上四段结构必须稳定可用。

### 3.4 表单引用解析

- 节点配置表单时 `configured=true`
- 成功解析时 `resolved=true`
- 未配置表单不影响流程处理
- 解析失败不阻断流程流转

### 3.5 统一流程图入口

- `GET /api/process-instances/{id}/diagram-view` 可用
- 单流程返回 `mode=single`
- 父子流程返回 `mode=composite`
- 新前端只依赖 `diagram-view`
- 兼容接口仍可正常工作

## 4. 真实业务回归要求

正式版前至少要完成以下真实业务回归。

### 4.1 文物鉴定流程

至少覆盖：

- 发起
- 待办处理
- 已办查询
- 任务上下文
- 表单引用解析
- 流程图展示
- 时间线展示

### 4.2 文物告警流程

至少覆盖：

- 发起
- 多角色流转
- 待办查询
- 已办查询
- 业务详情
- 流程图展示
- 时间线展示

### 4.3 父子流程复杂场景

至少覆盖：

- `callActivity` 直接子流程场景
- 单流程图与复合流程图共存
- `diagram-view` 自动判断逻辑

## 5. 当前兼容接口检查

虽然正式接口已经收口，但正式版前仍要确认兼容接口没有被误伤。

需要确认：

- `GET /api/process/instance/{id}/definition-xml` 仍可返回原始 XML
- `GET /api/process-instances/{id}/composite-diagram` 仍可返回统一视图结构

原因：

- 老系统仍可能依赖它们
- 正式版发布时需要给出兼容迁移窗口

## 6. 性能基线检查

正式版前不要求把所有大规模性能问题一次性彻底做完，但至少要确认当前版本在中小规模场景可稳定使用。

需要确认：

- 列表默认轻量化已生效
- 默认分页已生效
- 列表页不会高频逐条查完整进度
- `diagram-view` 只在详情场景调用
- Redis 缓存或投影不可用时有降级能力

## 7. 文档检查

正式版前，至少应完成以下中文文档：

- 《工作流平台标准接口说明》
- 《工作流接口收口说明》
- 《工作流配置说明》
- 《工作流业务接入指南》
- 《工作流前端接入交付模板》
- 《工作流后端接入交付模板》
- 《工作流测试验收模板》
- 《工作流版本变更说明》
- 《工作流正式版发布说明模板》
- 《工作流正式版发布检查结果》

## 8. 当前版本是否具备正式版条件

基于当前已完成内容，本文给出如下判断：

- 功能完整性：已基本达标
- 接口收口：已达标
- 单流程与复合流程图统一入口：已达标
- 两类真实业务联调：已达标
- 文档体系：已达标

结论：

- 当前版本已经具备正式版发布条件
- 正式发布时请同时附上《工作流正式版发布检查结果》

## 9. 发布动作建议

建议按以下顺序执行：

1. 冻结正式标准接口清单
2. 明确兼容接口清单
3. 整理正式版发布说明
4. 固定回归验证记录
5. 打正式版本标签

## 10. 建议正式版本号

建议当前这一轮正式版本号使用：

`v1.3.0`

版本含义：

- `1`：公共能力主版本稳定
- `3`：工作流平台能力已从接入验证走向正式交付
- `0`：首次正式稳定发布
