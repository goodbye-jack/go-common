# 配置模板使用说明

## 1. 新项目首次接入

推荐使用：

- `templates/releases/v1.3.1/config.initial.yaml`

作为业务项目真实 `config.yaml` 的初始化基线。

这个文件默认包含：

- `service_name`
- `addr`
- `databases.mysql.default`
- `databases.redis.default`
- `workflow` 基础骨架（默认关闭标准接口注册）

## 2. 为什么默认是 MySQL + Redis

当前模板默认使用：

- MySQL：作为默认关系型数据库
- Redis：作为常用依赖缓存

这并不意味着只支持 MySQL。

如果业务项目使用：

- PostgreSQL
- KingBase
- DM

请查看：

- `templates/releases/v1.3.1/config.latest.yaml`

并将其中对应数据库块替换真实 `config.yaml` 中的：

- `databases.mysql.default`

## 3. 为什么 Redis 仍默认生成

当前 `go-common` 中多处能力依赖 Redis，因此初始化模板默认保留：

- `databases.redis.default`

如果业务项目仍使用历史：

- `redis_addr`
- `redis_password`

请参考：

- `templates/releases/v1.3.1/config.compatibility.yaml`

## 4. latest 模板怎么用

`config.latest.yaml` 不建议直接用于运行。

它主要用于：

- 查看当前版本完整配置面
- 替换默认 MySQL 为其他关系型数据库
- 查阅 workflow / directory / assignment 的完整配置项

## 5. compatibility 模板怎么用

`config.compatibility.yaml` 仅用于：

- 老项目迁移参考
- 历史配置兼容说明

新项目不建议继续直接使用这些字段。
