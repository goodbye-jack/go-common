# 配置模板同步器说明

## 1. 定位

`configsync` 是 `go-common` 第二阶段新增的安全同步器，目标是让业务项目在**不改真实运行配置**的前提下，获得当前 tag 对应的标准配置产物。

当前实现边界：

- 自动识别业务项目真实 `config.yaml` 位置
- 真实配置不存在时，用 `config.initial.yaml` 初始化
- 始终生成同目录的 `config.latest.yaml`
- 按版本差异生成 `config.missing.yaml`
- 生成 `.go-common-config-meta.yaml`
- **默认不回写已有真实 `config.yaml`**

## 2. 路径识别顺序

同步器按以下顺序识别真实配置位置：

1. `.go-common-config-meta.yaml` 中记录的 `config_path`
2. 项目根目录 `config.yaml`
3. 项目根目录 `config.yml`
4. `config/config.yaml`
5. `config/config.yml`
6. 若均不存在：
   - 若存在 `config/` 目录，则选择 `config/config.yaml`
   - 否则选择根目录 `config.yaml`

生成的：

- `config.latest.yaml`
- `config.missing.yaml`
- `.go-common-config-meta.yaml`

都会写到真实配置所在目录。

## 3. 版本识别策略

同步器会读取业务项目 `go.mod` 中的：

- `require github.com/goodbye-jack/go-common vX.Y.Z`
- `replace github.com/goodbye-jack/go-common => ...`

并记录到 meta 中：

- `detected_version`
- `version_source`

说明：

- 若是本地 `replace` 但仍保留 `require v1.3.1`，则识别为 `v1.3.1`
- 若只有本地 `replace` 没有显式版本，则记录为 `local-dev`
- 模板生成目标版本默认使用当前库内置版本 `v1.3.1`

## 4. 缺失项生成规则

`config.missing.yaml` 只包含：

- 当前目标版本新增的配置项
- 且本地真实 `config.yaml` 中不存在的配置项

这里“存在”的判断只看**键路径是否存在**，不看值是否为空。

例如：

- `workflow.api.enabled: false` 已写在真实配置中 → 不会出现在 `config.missing.yaml`
- `workflow.directory.http.username:` 没写这个键 → 会出现在 `config.missing.yaml`

## 5. 对业务项目的意义

第二阶段完成后，业务项目至少可以获得：

- 首次接入时的标准初始配置
- 升级时的最新模板快照
- 升级时的缺失项补单
- 上次同步版本与真实配置路径记录

这为后续阶段继续做：

- 自动摘要日志
- `service.name/host/port` 兼容
- `databases.*.*.enabled`
- 发布期模板校验

提供了稳定基础。

## 6. 一行接入

若业务侧只是希望按默认规则生成产物，可直接调用：

```go
result, err := configsync.SyncProjectDefault(".")
```

默认行为：

- 配置不存在时自动初始化真实 `config.yaml`
- 生成同目录 `config.latest.yaml`
- 生成同目录 `config.missing.yaml`
- 生成同目录 `.go-common-config-meta.yaml`

## 7. 运行期安全自动同步

从当前版本开始，`go-common/config` 会在读取业务配置前尝试执行一次**安全自动同步**。

触发原则：

- 仅在当前进程所在目录向上能找到业务项目 `go.mod` 时触发
- 仅对依赖了 `github.com/goodbye-jack/go-common` 的业务项目触发
- 对 `go-common` 自身仓库不触发
- 若检测到生产环境标记，则默认跳过

自动行为：

- 真实 `config.yaml` 不存在时初始化
- 刷新 `config.latest.yaml`
- 刷新 `config.missing.yaml`
- 刷新 `.go-common-config-meta.yaml`
- **绝不覆盖已有真实 `config.yaml`**

生产环境规避：

- 当 `GO_ENV` / `APP_ENV` / `RUN_ENV` / `CONFIG_ENV` 任一值为 `prod` 或 `production` 时，默认跳过

手动开关：

- `GO_COMMON_CONFIG_SYNC=off`：关闭自动同步
- `GO_COMMON_CONFIG_SYNC=force`：忽略生产环境标记，强制自动同步
